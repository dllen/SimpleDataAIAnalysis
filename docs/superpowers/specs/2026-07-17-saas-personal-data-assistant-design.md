# 个人数据问答助手 SaaS 产品设计与项目计划

## 1. 产品概述

### 1.1 产品定位

面向非专业用户的**个人数据问答助手**。用户上传表格数据后，可直接用自然语言获得分析结论、图表和可执行建议，而不是先学习 SQL、透视表或 BI 工具。

### 1.2 核心价值

- 降低数据分析门槛：不用写 SQL，不用手动整理复杂报表。
- 提升决策速度：上传文件后立刻得到结论，而不是先花半小时处理格式。
- 降低数据风险：明确展示多用户隔离、只读执行沙箱、清洗可回滚。

### 1.3 目标用户

- 职场新人、运营、销售、财务、HR 等经常处理 Excel 的岗位。
- 学生、研究者、课程助手，需要快速理解 survey、实验或课程数据。
- 小个体户、副业者、内容创作者，需要快速看懂订单、库存、投放或选题数据。

### 1.4 成功指标

- 新用户 10 秒内完成第一次有效分析。
- 免费用户 7 日内转化率、Pro 订阅留存率。
- 用户平均对话轮数、会话保存率、导出/分享率。

## 2. 市场与竞争分析

### 2.1 市场机会

大量普通用户每周都在处理表格，但专业数据分析工具学习成本高。通用 AI 聊天机器人虽然能回答问题，却很难直接操作本地文件、执行结构化查询、返回可视化结果。这之间正好存在一个**轻量自助分析 SaaS** 的空白市场。

### 2.2 竞品对标

- ChatGPT / 通用 AI：懂自然语言，但不擅长直接基于用户上传文件做稳定结构化分析。
- Excel / WPS：熟悉，但需要用户手动做透视表、清洗、公式。
- 传统 BI / 低代码报表：功能强，但配置复杂，不适合个人轻量场景。
- 垂直 AI 数据工具：概念接近，但普遍存在“注册门槛高、结果不稳定、不够中文友好”的问题。

### 2.3 差异化策略

本项目不主打“最强大的分析引擎”，而主打**最省时间、最敢传文件、最像 Excel 升级版**的体验：

- 上传即分析，不用复杂配置。
- 自然语言提问，直接返回结论 + 表格 + 图表。
- 显性展示数据安全能力，让敏感表格用户敢用。
- 自动识别脏数据，并给出可执行清洗建议。

## 3. 商业模式

### 3.1 收费模式

采用订阅制 + 额度制：

- 免费版：体验核心能力，适合试用和小文件低频使用。
- Pro 版：按月订阅，解锁更高上传额度、更长对话、导出和保存能力。
- 团队版：后续扩展，初期不急于上线。

### 3.2 最小付费闭环

先跑通：免费体验 -> 注册 -> 上传真实文件 -> 完成分析 -> 触发 Pro 限制 -> 订阅。

优先验证的是：**有人愿意为“更方便分析自己的文件”付小钱**，而不是一开始就做企业级功能。

### 3.3 定价方向

- 免费版：有限文件大小、有限对话条数、基础预览和导出。
- Pro 版：更大文件、更多会话、高级导出、更长的上下文、优先处理。
- 不靠广告，靠产品价值转化。

## 4. 产品功能设计

### 4.1 V1：核心体验版

目标：让用户感受到“上传表格就能分析数据”的核心价值。

核心功能：

- 游客演示数据体验：打开即可提问，无需先注册。
- 上传 CSV / Excel / JSON 并自动识别字段。
- 自然语言分析，返回结论摘要、结果表格、图表。
- 基础导出：导出结果、图表或会话。
- 登录注册与基础用户数据隔离。
- 订阅/额度体系的最小版本。

### 4.2 V2：助手感增强版

目标：提升留存和付费转化，让工具更像“数据助手”而不是“查询器”。

核心功能：

- 上传后自动推荐 3-5 个最可能想问的问题。
- 分析结果自动生成自然语言结论，不只是图表。
- 自动发现脏数据，给出清洗建议并支持一键执行。
- 支持保存分析会话和清洗结果。
- 更丰富的可视化类型和结果页布局。

### 4.3 V3：留存与增长版

目标：提升复访、分享和口碑传播。

核心功能：

- 场景模板库：订单分析、survey 分析、财务核对、学习记录分析。
- 分享/导出工作流：把分析过程整理成可复用的结果包。
- 新手引导与产品内教程。
- 更稳定的多模型切换、使用偏好保存。
- 基础运营看板：转化、留存、会话质量。

## 5. 用户旅程

### 5.1 新用户旅程

1. 打开产品，先看到演示数据。
2. 直接问一个示例问题，立刻看到结论和图表。
3. 感受到“这能帮我解决实际问题”。
4. 注册/登录。
5. 上传自己的文件。
6. 看到自动推荐问题，完成第一次真实分析。
7. 被引导保存会话、继续追问、或升级到更高额度。

### 5.2 付费用户旅程

1. 日常处理 Excel 时想起这个产品。
2. 上传文件并完成分析。
3. 遇到文件大小、导出、保存等限制时，自然看到 Pro 价值。
4. 订阅后获得更稳定、更高效的使用体验。
5. 长期留存来自“每周都有一点数据要处理”的真实需求。

## 6. 增长策略

### 6.1 产品驱动增长

- 结果页支持导出和保存，提升用户离开前带走价值。
- 让免费用户完整跑通一次真实分析，而不是只看演示。
- 减少注册阻力：游客体验 -> 注册 -> 上传 -> 分析 -> 升级。

### 6.2 内容与传播

- 制作“把 Excel 扔进去就能问 AI”的短视频和图文案例。
- 围绕高频场景做内容：职场报表、survey 分析、订单复盘、学习数据。
- 提供模板和案例文件，降低首次使用门槛。

### 6.3 转化策略

- 在分析结果页提示升级点，而不是在首页硬推订阅。
- 限制点放在“用户最需要继续完成工作时”才出现。
- 先做小金额订阅，验证付费意愿后再扩功能。

## 7. 技术架构与演进

### 7.1 现有优势

本项目已经具备较完整的数据分析内核：

- Spring Boot + Spring AI + DuckDB 的核心链路。
- 多用户数据隔离与 JWT 认证。
- SQL 只读沙箱与安全执行边界。
- 数据清洗检测、清洗建议、清洗历史记录。
- SSE 流式输出，改善对话体验。

### 7.2 V1 技术重点

- 新增游客演示态与演示数据支持。
- 结果层增加 AI 摘要，而不是只返回原始结果。
- 增加订阅额度、限流和导出能力。
- 完善会话保存与继续追问能力。

### 7.3 V2-V3 技术重点

- 模板系统、推荐问题生成、结果分享。
- 更丰富的可视化、导出格式和会话管理。
- 运营数据埋点与转化漏斗分析。
- 稳定性、模型切换、错误提示和降级策略。

## 8. 运营与财务规划

### 8.1 运营重点

前期不要依赖大规模投放，优先做：

- 产品内转化和自然传播。
- 高频场景模板和真实案例。
- 内容营销与社群反馈。

### 8.2 财务假设

初期重点不是利润最大化，而是验证：

- 单个获客成本是否可控。
- 免费到付费转化率是否成立。
- 用户生命周期价值是否大于获客成本。

建议先设定一个小型增长目标，例如：首月获取种子用户并跑通 10-50 笔真实付费，再决定是否扩大投入。

### 8.3 关键经营假设

- 用户愿意为“节省 15-30 分钟的数据整理时间”支付小额月费。
- 高频表格用户会在多个月份重复使用。
- 中文自然语言分析是主要卖点，不是英文 prompt 能力。

## 9. 风险与应对

- LLM 结果不稳定：通过摘要化、推荐追问、人工可干预 SQL、错误提示缓解。
- 用户担心数据安全：显性展示隔离、沙箱、清洗回滚能力。
- 竞品快速跟进：靠场景模板、产品体验、中文场景深度建立壁垒。
- 付费转化低：先做最小闭环验证，不要一开始就做复杂套餐。
- 成本过高：优先控制模型调用成本，缓存常见分析，限制演示和免费额度。

## 10. 落地节奏

### Phase 1：验证价值

- 游客演示体验。
- 上传 + 自然语言分析 + 基础可视化。
- 注册 + 登录 + 基础隔离。
- 最小额度/订阅能力。

### Phase 2：提升留存

- 自动推荐问题。
- AI 结论摘要。
- 自动清洗建议与执行。
- 会话保存与继续追问。

### Phase 3：规模增长

- 场景模板库。
- 分享/导出工作流。
- 新手引导与运营看板。
- 更稳定的增长和商业化迭代。
